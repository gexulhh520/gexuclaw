给大模型开发的最终修改说明
目标

把 artifact 产物声明机制改成：

是否需要 artifact，不再由 AgentVersion.contextPolicyJson 控制。
只由 TaskEnvelope.expectedResult.kind 决定。

当 expectedResult.kind === "artifact"：
- Runtime 强制开启 artifact directives full mode
- 子 Agent 必须输出 <artifact_directives>...</artifact_directives>
- <artifact_directives> 内部必须是严格 JSON
- 至少包含一个 declaredArtifacts

  args.input.taskEnvelope

1.3 新增函数

放在 AgentRuntime class 外部：

function resolveArtifactDirectiveConfigFromTaskEnvelope(
  taskEnvelope?: TaskEnvelope
): ArtifactDirectiveConfig {
  if (taskEnvelope?.expectedResult.kind === "artifact") {
    return {
      enabled: true,
      mode: "full",
    };
  }

  return {
    enabled: false,
    mode: "decision_only",
  };
}

5. <artifact_directives> JSON 规范

系统只解析 tag 内部的 JSON。格式如下：

普通 final answer

<artifact_directives>
{
  "declaredArtifacts": [
    {
      "artifactType": "structured_data",
      "artifactRole": "output",
      "title": "产物标题",
      "contentJson": {}
    }
  ]
}
</artifact_directives>
5.1 declaredArtifacts 字段
{
  artifactType: "text" | "structured_data" | "page" | "image" | "link" | "file" | "collection";
  artifactRole: "input" | "reference" | "intermediate" | "draft" | "final" | "output" | "pending_write";
  title: string;

  contentText?: string;
  contentJson?: Record<string, unknown> | unknown[];
  uri?: string;
  mimeType?: string;
  metadata?: Record<string, unknown>;
  sourceArtifactIds?: string[];
}

必须字段：

artifactType
artifactRole
title
