# 前端消息内容回显优化计划

## 问题描述

当前前端聊天界面中，用户发送的消息 `content` 可能包含多种类型（text、image、audio、document），但 `formatMessageContent` 函数只处理了 text、image、audio 三种类型，导致 document 类型无法正确回显。

示例数据：
```json
[
  {"type": "text", "content": "这个文档说了什么"},
  {"type": "document", "content": "/documents/2026/04/d86df89660534e30ad9d5d8577182a78.docx"}
]
```

## 目标

修改 `formatMessageContent` 函数，使其能够正确处理和回显所有类型的消息内容，特别是 document 类型。

## 实现步骤

### 步骤 1: 修改 formatMessageContent 函数

**文件**: `frontend/src/views/Chat.vue`

**修改内容**:
在 `formatMessageContent` 函数中添加对 `document` 类型的处理：

```typescript
function formatMessageContent(content: any): string {
  if (typeof content === 'string') {
    return content
  }
  
  if (Array.isArray(content)) {
    return content.map((item: any) => {
      if (item.type === 'text') {
        return item.content
      } else if (item.type === 'image') {
        return `[图片: ${item.content}]`
      } else if (item.type === 'audio') {
        return `[音频: ${item.content}]`
      } else if (item.type === 'document') {
        // 提取文件名
        const filename = item.content.split('/').pop() || item.content
        return `[文档: ${filename}]`
      }
      return ''
    }).join('\n')
  }
  
  return String(content)
}
```

### 步骤 2: 优化文档显示样式（可选增强）

如果需要更好的视觉效果，可以将文档显示为可点击的链接：

```typescript
} else if (item.type === 'document') {
  const filename = item.content.split('/').pop() || item.content
  return `<a href="${item.content}" target="_blank" class="document-link">📄 ${filename}</a>`
}
```

同时添加对应的 CSS 样式：


## 验收标准

1. 当消息 content 包含 document 类型时，能够正确显示文档文件名
2. 文档显示格式统一，易于识别
3. 不影响其他类型（text、image、audio）的显示
4. 向后兼容，处理未知类型时不会报错
