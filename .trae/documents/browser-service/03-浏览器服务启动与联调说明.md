## 一、当前问题

虽然浏览器独立服务骨架已经实现，但当前运行链路仍可能继续走本地模式，原因通常是：

1. `backend/.env` 未开启 `BROWSER_SERVICE_ENABLED=true`
2. 浏览器独立服务进程未启动
3. Celery Worker 未重启，仍缓存旧配置

---

## 二、建议环境变量

在 `backend/.env` 中增加：

```env
BROWSER_SERVICE_ENABLED=true
BROWSER_SERVICE_URL=http://127.0.0.1:8001
BROWSER_SERVICE_HOST=127.0.0.1
BROWSER_SERVICE_PORT=8001
```

---

## 三、启动命令

### 3.1 启动浏览器服务

在仓库根目录执行：

```bash
python -m uvicorn backend.browser_service.app:app --host 127.0.0.1 --port 8001
```

### 3.2 健康检查

```bash
curl http://127.0.0.1:8001/health
```

期望返回：

```json
{
  "status": "ok",
  "service": "browser-service"
}
```

### 3.3 重启 Celery Worker

因为工具运行时和配置会在 Worker 启动时加载，开启服务模式后需要重启 Celery Worker，确保读取到新的环境变量。

## 四、联调顺序

1. 修改 `backend/.env`
2. 启动浏览器服务
3. 健康检查通过
4. 重启 API / Celery Worker
5. 重新触发 `browser__launch_browser`
6. 检查日志中不再出现本地 `TaskBrowserManager` 启动记录

---

## 五、成功标志

联调成功后，应满足：

1. Agent Worker 不再直接打印本地 `TaskBrowserManager` 启动浏览器日志
2. `BrowserTool` 调用走 HTTP
3. `launch_browser` 不再在 Worker 进程内触发 `create_subprocess_exec -> NotImplementedError`
