# 保留原始文件名计划

## 问题描述

当前文件上传后，使用 UUID 生成新的文件名保存，导致：
1. 前端回显时显示的是 UUID 文件名（如 `d86df89660534e30ad9d5d8577182a78.docx`）
2. 用户无法识别文件内容
3. 原始文件名信息丢失

当前代码：
```python
def _generate_filename(original_filename: str) -> str:
    """Generate unique filename with UUID"""
    ext = original_filename.rsplit(".", 1)[-1] if "." in original_filename else ""
    return f"{uuid.uuid4().hex}.{ext}" if ext else uuid.uuid4().hex
```

## 目标

修改文件上传逻辑，保留原始文件名，同时确保文件唯一性（避免同名文件冲突）。

## 实现方案

### 方案：保留原始文件名 + UUID 前缀

文件保存格式：`{UUID}_{原始文件名}`

例如：
- 原始文件：`矩阵式监理专业人才TOP榜评选实施方案.docx`
- 保存文件名：`d86df89660534e30ad9d5d8577182a78_矩阵式监理专业人才TOP榜评选实施方案.docx`
- 回显时提取：`矩阵式监理专业人才TOP榜评选实施方案.docx`

### 实现步骤

#### 步骤 1: 修改 `_generate_filename` 函数

**文件**: `backend/api/upload.py`

```python
def _generate_filename(original_filename: str) -> str:
    """Generate unique filename while preserving original name"""
    ext = original_filename.rsplit(".", 1)[-1] if "." in original_filename else ""
    name = original_filename.rsplit(".", 1)[0] if "." in original_filename else original_filename
    # 清理文件名中的特殊字符，保留中文、英文、数字、空格、下划线、连字符
    import re
    safe_name = re.sub(r'[^\w\s\-\u4e00-\u9fff]', '_', name)
    # 限制文件名长度，避免过长
    if len(safe_name) > 100:
        safe_name = safe_name[:100]
    unique_id = uuid.uuid4().hex[:8]  # 使用短UUID前缀
    if ext:
        return f"{unique_id}_{safe_name}.{ext}"
    return f"{unique_id}_{safe_name}"
```

#### 步骤 2: 修改前端 `formatMessageContent` 函数

**文件**: `frontend/src/views/Chat.vue`

提取保存文件名中的原始文件名部分：

```typescript
} else if (item.type === 'document') {
  // 从格式 "{UUID}_{original_name}.{ext}" 中提取原始文件名
  const fullname = item.content.split('/').pop() || item.content
```

## 备选方案

### 方案 B：在 UploadResponse 中返回原始文件名

修改 `UploadResponse` 模型，增加 `original_filename` 字段：

```python
class UploadResponse(BaseModel):
    ...
    filename: str           # 保存的文件名（带UUID）
    original_filename: str  # 原始文件名
    ...
```

前端直接使用 `original_filename` 显示。

**优点**：
- 逻辑清晰，不依赖文件名格式解析
- 灵活性高，可以随时调整存储格式

**缺点**：
- 需要修改前后端接口
- 需要存储额外的原始文件名信息

## 推荐方案

**推荐方案 A**（文件名前缀方案）：
- 改动最小
- 向后兼容（旧文件仍然可以正常显示，只是没有原始文件名）
- 不需要修改接口

## 验收标准

1. 文件上传后保存格式为 `{UUID}_{原始文件名}.{ext}`
2. 前端回显时显示原始文件名（不含 UUID 前缀）
3. 支持中文文件名
4. 处理特殊字符（替换为下划线）
5. 文件名过长时自动截断

## 涉及文件

- `backend/api/upload.py` - 修改文件名生成逻辑
- `frontend/src/views/Chat.vue` - 修改文件名提取逻辑

## 风险评估

- **风险等级**: 低
- **影响范围**: 仅影响新上传的文件，旧文件不受影响
- **回滚方案**: 恢复 `_generate_filename` 函数到修改前状态
