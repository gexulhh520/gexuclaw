## 一、本次已完成

### 1. 文档归档

- 已将知识库相关文档统一归档到 `.trae/documents/knowledge-base/`
- 已补齐导航、总方案、API 契约、开发任务清单

### 2. 后端模型

- 新增 `KnowledgeBase` 和 `KnowledgeDocument` 模型
- 为 `User` 增加知识库和知识库文档关联
- 为 `ChatSession` 增加 `knowledge_base_ids` 字段

### 3. 后端 Schema

- 新增知识库创建、更新、响应 Schema
- 新增知识库文档响应 Schema
- 扩展会话 Schema，支持 `knowledge_base_ids`

### 4. 后端服务层

- 新增知识库服务
- 支持知识库创建、列表、详情、更新、删除
- 支持知识库文档列表查询
- 支持基础统计字段 `document_count` 和 `chunk_count`

### 5. 后端接口

- 新增知识库路由
- 支持知识库 CRUD
- 支持查看某个知识库的文档列表
- 支持更新会话绑定的知识库列表
- 会话创建和聊天请求已接入 `knowledge_base_ids`

### 6. 基础验证

- 已通过编辑器诊断检查
- 已通过 `python -m py_compile` 轻量编译校验

### 7. 第二阶段后端链路

- 新增知识库文档上传接口
- 新增知识库文档删除接口
- 文档上传后立即写入 `knowledge_documents`
- Celery 任务已支持回写 `processing / success / failed`
- 知识检索已支持 `knowledge_base_ids` 过滤
- Worker 到 Agent 工具链路已传递 `knowledge_base_ids`
- 知识库文档删除、知识库删除时会清理原文件和 LanceDB 分块
